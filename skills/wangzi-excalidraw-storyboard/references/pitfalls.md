# Common mistakes

Avoid:

- Drawing before fetching URL content when the input was link-only.
- Keeping the briefing internal only (user must get visible helper text).
- Assuming MCP installs itself without guidance or verification.
- Shipping a board with no title or label.
- Putting speaker directions on the audience-facing canvas.
- Using portrait talking-head layout but leaving **no** margin for bottom-right face cam when that layout was intended.
- Calling `create_view` with no visible result in chat and skipping the MCP export that yields a browser share link.
